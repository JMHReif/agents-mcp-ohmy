= Agents, Tools, and MCP, oh my!

This repository demonstrates a *progressive reveal* of AI agent architecture:

* *Act 1* — LLM Only: The brain alone (hallucinates without data access)
* *Act 2* — Spring AI Tools: The brain gets hands (deterministic, structured results)
* *Act 3* — Memory: Conversation context backed by Neo4j (follow-up questions, chained reasoning)
* *Act 4* — MCP Protocol: Standardized tool discovery (any query, dynamic capabilities)

Each act builds on the previous, showing _why each architectural layer becomes necessary_.

== Running the application

Set environment variable (if applicable) for Large Language Model - e.g. OpenAI.

[source,shell]
----
source ./env.sh
----

Run the application.

[source,shell]
----
./mvnw spring-boot:run
----

== Demo Script

=== Act 1: LLM Only — "The Brain Alone"

The LLM has no tools. It will hallucinate or admit it can't access data.

[source,bash]
----
# Ask about data it can't access
http ":8080/act1/query?userQuery=What are the highest rated books?"

# Ask for personalized recommendations
http ":8080/act1/query?userQuery=What books should I read next?"
----

*Key point:* The LLM is smart but blind. It has no access to real data.

---

=== Act 2: Tools — "The Brain Gets Hands"

Same questions, but now the agent has `@Tool` methods that can query Neo4j.

[source,bash]
----
# Same question — now tools execute and return real data
http ":8080/act2/query?userQuery=What are the highest rated books?"

# Personalized recommendations (tools query the graph)
http ":8080/act2/query?userQuery=Could you find well-rated books for me to read?"

# Count books read
http ":8080/act2/query?userQuery=How many books have I read?"

# Multi-hop graph traversal: author's other works
http ":8080/act2/query?userQuery=I loved Dune. What else did that author write?"

# Multi-hop graph traversal: collaborative filtering
http ":8080/act2/query?userQuery=Recommend books based on what readers with similar taste enjoyed"

# RAG: semantic search on reviews
http ":8080/act2/query?userQuery=Find book reviews where readers changed their perspective"

# GraphRAG: semantic search + graph enrichment
http ":8080/act2/query?userQuery=I want to find fantasy books about dragons and magic"
----

*Key point:* Tools give the LLM deterministic access to real data. Watch the logs to see tool execution (`>>> TOOL CALLED` / `<<< TOOL RESULT`).

---

=== Act 3: Context with memory — "The Brain Remembers"

Everything from Act 2, plus conversation memory stored in Neo4j. Follow-up questions reference prior results.

[source,bash]
----
# First question — establishes context
http ":8080/act3/query?userQuery=What books should I read next based on what readers with similar taste enjoyed?&conversationId=demo1"

# Another follow-up — chained reasoning across the conversation
http ":8080/act3/query?userQuery=Who are the authors of those books?&conversationId=demo1"

# Start a fresh conversation (different conversationId)
http ":8080/act3/query?userQuery=What books should I read next based on what readers with similar taste enjoyed?&userId=0007f8dd09337afd986d765569cf0110&conversationId=demo2"
----

*Key point:* The agent now chains reasoning across prior results. Context + Graph + Agent loop — all stored in Neo4j.

=== Act 4: MCP Protocol — "The Standard"

Same capabilities, but tools are now discovered dynamically via MCP.

[source,bash]
----
# Tool discovery — what tools are available?
http ":8080/act4/debug/tools"

# Ask the agent about its tools
http ":8080/act4/query?userQuery=What tools do you have access to?"

# Schema-aware querying — agent discovers the schema first
http ":8080/act4/query?userQuery=What does the database schema look like?"

# Now the agent can write ANY Cypher query
http ":8080/act4/query?userQuery=What are the highest rated books?"

# Graph-native queries the agent constructs on the fly
http ":8080/act4/query?userQuery=Which authors wrote the most books?"
http ":8080/act4/query?userQuery=What is the leaderboard for number of books each user has read?"
http ":8080/act4/query?userQuery=Find authors whose books have the most diverse set of reviewers"
----

*Key point:* We didn't rewrite business logic. We changed the integration layer. MCP standardizes tool access.

---

== Architecture Overview

[source]
----
Act 1: User → LLM → (hallucination)
Act 2: User → LLM → @Tool methods → Neo4j Repository → Neo4j
Act 3: User → LLM → Context Memory → Neo4j (queries + conversation history)
Act 4: User → LLM → MCP Protocol → Neo4j MCP Server → Neo4j
----
